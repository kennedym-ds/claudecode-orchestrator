# /spec — Specification Builder

Build a specification for: $ARGUMENTS

## Instructions

1. Start with the 5-phase elicitation dialogue (discovery → scope → behavior → risk → assembly)
2. Ask clarifying questions (max 5 per phase) before drafting
3. Define actors, user stories, acceptance criteria (Given/When/Then)
4. Identify edge cases, constraints, and non-functional requirements
5. Generate the spec in the requested format (Markdown, Jama JSON, or Confluence wiki)
6. Output to `artifacts/plans/` or the requested destination

Use the spec-builder agent with spec-elicitation skill.
