# /test-arch — Test Architecture

Design the test strategy for: $ARGUMENTS

## Instructions

1. Analyze the codebase for testability and existing coverage
2. Design the test pyramid: unit → integration → e2e ratio
3. Identify coverage gaps and critical paths lacking tests
4. Recommend test infrastructure (frameworks, fixtures, CI integration)
5. Define test data strategy: factories, seeds, anonymized snapshots
6. Produce a test plan with priority-ordered test cases

Use the test-architect agent with tdd-workflow and verification-loop skills.
