# /e2e — End-to-End Testing

Write end-to-end tests for: $ARGUMENTS

## Instructions

1. Identify the critical user flows to test
2. Choose the appropriate framework (Playwright, Cypress, supertest, etc.)
3. Write acceptance tests covering happy path and key error paths
4. Include setup/teardown for test data and environment
5. Make tests deterministic — no flaky waits, use proper selectors
6. Validate both functional behavior and key non-functional aspects (performance, accessibility)

Use the e2e-tester agent with verification-loop skill.
