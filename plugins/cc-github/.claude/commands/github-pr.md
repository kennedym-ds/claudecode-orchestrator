# /github-pr — Pull Request Workflow

Manage a pull request: $ARGUMENTS

## Instructions

1. If creating: analyze branch diff, generate title/body/labels
2. If reviewing: fetch PR, categorize risk, flag issues, summarize
3. If merging: check CI status, verify approvals, merge strategy
4. Use conventional commits for PR titles
5. Link related issues automatically

Use the github-pr agent with pr-workflow skill. Requires GitHub MCP server.
